# Bài 2.3 — Khởi tạo project multi-module Maven + Spring Boot

> Ghi lại **nguyên văn** nội dung buổi học: lý thuyết + thực hành + kết quả thật.

## 1. Vì sao multi-module Maven?

Hệ có 5 service + 1 module contract dùng chung. Maven **multi-module** cho phép:
- **Một parent POM** quản version tập trung (Java, Spring Boot, Kafka, Avro) → mọi module **cùng version**, tránh lệch version.
- **Module contract dùng chung** (`common-events`) build một lần, service phụ thuộc vào → tất cả nói **cùng "hợp đồng" Avro** (Bài 0.2).
- Build/test toàn hệ một lệnh, nhưng mỗi service vẫn là **Spring Boot app độc lập** (main riêng, port riêng, DB riêng).

```
source/
├── pom.xml                 ← PARENT POM (aggregator + quản version)
├── mvnw / mvnw.cmd / .mvn  ← Maven wrapper (build tái lập, không cần Maven cài sẵn)
├── common-events/          ← contract Avro dùng chung (Module 3 thêm schema)
├── account-service/        ← Spring Boot app đầu tiên (template)
└── (4 service còn lại thêm dần khi tới bài của nó)
```

## 2. Hai khái niệm Maven cần phân biệt
- **Parent (inheritance)**: con *kế thừa* version/cấu hình từ cha (`dependencyManagement`, properties). Cha của ta kế thừa `spring-boot-starter-parent` để hưởng BOM Spring Boot.
- **Aggregator (`<modules>`)**: cha *gom* các module con build cùng lúc. POM của ta vừa parent vừa aggregator.

## 3. ⚠️ Mono-repo vs Multi-repo (điểm khác prod đã nêu rõ)
> Dùng **mono-repo** (tất cả service một repo, một parent POM) — **lựa chọn cho việc học**: gọn, dễ build, dễ theo dõi.
>
> **Prod thật**: microservices thường tách **mỗi service một repo riêng** (deploy/versioning/pipeline độc lập — "independent deployment" Bài 0.1). Đánh đổi: mono-repo dễ học nhưng dễ vô tình tạo coupling; multi-repo đúng triết lý hơn nhưng nặng vận hành.
>
> → Với mục tiêu học, mono-repo hợp lý. Đây là điểm khác prod cần ý thức.

## 4. Các file đã tạo

**`source/pom.xml`** (parent): kế thừa `spring-boot-starter-parent:3.4.1`, `java.version=21`, khai báo `<modules>`, `dependencyManagement` cho Confluent (`kafka-avro-serializer`, `avro`), thêm **repo Confluent** (kafka-avro-serializer không có trên Maven Central).

**`source/common-events/pom.xml`**: thư viện thuần (`packaging=jar`, KHÔNG phải Spring Boot app), chỉ có dependency `avro` core. Module 3 sẽ thêm schema `.avsc` + `avro-maven-plugin`.

**`source/account-service/`**:
- `pom.xml`: `spring-boot-starter-web` (REST — command/query vào), `spring-kafka` (event), `spring-boot-starter-actuator` (health/metrics — chuẩn bị Module 8), `common-events`, test deps, `spring-boot-maven-plugin` (đóng gói executable jar).
- `AccountServiceApplication.java`: `@SpringBootApplication` main class.
- `application.yml`: `server.port=8090` (tránh 8080=Kafka UI, 8081=Schema Registry); `spring.kafka.bootstrap-servers=localhost:19092,19093,19094` (listener EXTERNAL — Bài 2.1).

> 📌 Quy ước port các service (tránh đụng 8080/8081):
> account=8090, transaction=8091, ledger=8092, fraud=8093, notification=8094.

## 5. ✅ Kết quả thực hành (đã verify)

**Generate Maven wrapper:** `mvn -N wrapper:wrapper` → tạo `mvnw`, `mvnw.cmd`, `.mvn/wrapper/`.

**Build:** `./mvnw -DskipTests clean package` → thành công:
```
45M   account-service/target/account-service-0.0.1-SNAPSHOT.jar   (fat executable jar)
1.8K  common-events/target/common-events-0.0.1-SNAPSHOT.jar       (thư viện thuần)
```

**Lỗi gặp & cách sửa (bài học XML):** ký tự `&` thô trong `<description>` làm POM non-parseable
(*"entity reference names can not start with character"*). XML yêu cầu escape `&amp;` (hoặc tránh `&`). Đã sửa.

**Smoke test:** `java -jar account-service-...jar` → boot thành công:
```
Tomcat started on port 8090 (http)
Started AccountServiceApplication in 3.053 seconds
GET /actuator/health → {"status":"UP", ...}
```
